### Запустить
После установки репозитория есть 2 способа запуска программы:

### Способ №1: с использованием Dockerfile

Соберите образ: ```docker build -t lab1_python .```

Запустите: ```docker run -it --rm lab1_python```

Вы увидите приветственное сообщение с инструкцией, но информация оттуда продублирована и тут. Теперь вы можете запускать тесты или вводить команды для самой программы.

- Для обычного запуска тетсов введите: ```pytest```
- Для запуска тестов с информацией о покрытии введите: ```pytest --cov=src/ --cov-report=term```
- Чтобы посмотреть подробную инструкцию к командам самой программы с примерами введите: ```python -m src instruct```
- Команды для самой программы вводите согласно этой инструкции. В разделе "Синтаксис" ниже также есть та же самая инструкция.
- Для выхода из программы введите: ```exit``` или ```quit```

### Синтаксис
Т.к. интерфейс программы создан с помощью typer, при каждом вводе команды нужно заново запукать программу, поэтому в начале каждой команды будет ```python -m src```. 
В самой программе доступны следующие 3 команды:

1) instruct - Выводит подобное сообщение с интрукцией
   У этой команды нет опций или аргументов
   Пример использования: python -m src instruct

2) sources - Выводит список доступных источников задач
   У этой команды нет опций или аргументов
   Пример использования: python -m src sources

3) read - Читает и показывает задачи из указанных источников с возможностью фильтрации, имеет несколько опций и аргументов:
В качестве аргументов можно указать файлы формата jsonl, тогда задачи будут читаться из них
        Примеры:
        python -m src read example_1.jsonl
        python -m src read example_1.jsonl example_2.jsonl
        В самом репозитории уже есть 2 jsonl файла (example_1.jsonl и example_2.jsonl), с котрыми вы можете протестировать             работу программы самостоятельно.
--stdin - Считывать задачи из сдандартного ввода
        Задачи нужно писать в формате id:title:author:content
        Если вы захотите перестать вводить задачи, напишите stop
        Пример использования:
        python -m src read --stdin
        Дальше вводим задачи, например:   my_id:my_title:my_author:my_content
--max_tasks - Установить максимум, сколько задач может быть прочитано
        По умолчанию 10, если поставить 0, то можно прочитать неограниченное число задач.
        При одновременном вводе --max_tasks 0 и --stdin не забывайте про возможность завершить ввод задач командой stop.
        Примеры:
        python -m src read --stdin --max-tasks 5
        python -m src read example_1.jsonl --max-tasks 0
--author - Считывать задачи только указанного автора
        Пример:
        python -m src read --stdin --author milena
        Все встреченные задачи с другим автором будут проигнорированы и не будут считываться программой
--contains - Считывать только задачи с указанной подстрокой в содержании
        Пример:
        python -m src read example_1.jsonl --contains проект
        Задача '8:Сделать бэкап:milena:Сделать бэкап базы данных проекта' будет считана
        Задача '9:Написать тесты:cool_user:Написать тесты к лабе по python' не считается
Все аргументы и опции можно комбинировать, примеры:
python -m src read example_1.jsonl example_2.jsonl --max-tasks 20 --author kate
python -m src read example_1.jsonl example_2.jsonl --stdin --max-tasks 20
python -m src read example_1.jsonl --stdin --author alina --contains python
