## Описание решения
В этой ЛР оснвной упор делаля на работу с источниками и идентичную логику обработки задач независимо от её источника. Опишу основные компоненты решения:
### Основные классы
* Класс Task
  
  Описывает задачу как набор данных: id, название, автор и содержание. Реализован с помощью dataclass c дополнительными параметрами frozen=True и slots=True, которые защищают задачу от изменений после создания и не позволяют давать ей никаких других параметров, кроме заранее заданных.
  
* TaskSource
  
  Описан с помощью typing.Protocol. Является протоколом, который описывает поведение источника задач. Он говорит о том, что у источника должно быть название и метод чтения задач. Вместе с этим классом используется декоратор @runtime_checkable (runtime-проверка соблюдения контракта), благодаря чему можно применять isinstance() для проверки выполнения контракта. Все это делает возможным не связывать источники наследованием и добавлять новые источники без изменения старого кода, и демонстрирует реальное использование котрактного программирования.
  
* TaskManager
  
  Вспомогательный класс, который умеет проходится по всем источникам и возвращать задачи от всех из низ. Опять же, задачи обрабатываются идентично вне зависимости от своего источника.
### Конкретные источники
Я реализовала 2 конкретных источника задач:
* JSON файлы
  
  Для этого типа источников был создан класс JsonlSource, где есть метод get_tasks для парсинга задач из файла формата jsonl и их выдачи. Т.е. этот класс соответствует протоколу источника, заданному раннее.
  
* stdin

  Для этого источника был создан класс StdinLineSource, реализующий метод get_tasks для парсинга задач из ввода пользователя. Он также полностью соответствует протоколу источника.
  
### cli-интерфейс

Для создания cli-интерфейса в этой ЛР была использована бибилотека typer. Этот интерфейс просто позволяет продемонстрировать обработку задач из всех добавленных источников, его команды будут описаны ниже, в разделе "Синтаксис".

### Синтаксис
Т.к. интерфейс программы создан с помощью typer, при каждом вводе команды нужно заново запукать программу, поэтому в начале каждой команды будет ```python -m src```. 
В самой программе доступны следующие 3 команды:

1) instruct - Выводит подобное сообщение с интрукцией
   У этой команды нет опций или аргументов
   Пример использования: python -m src instruct

2) sources - Выводит список доступных источников задач
   У этой команды нет опций или аргументов
   Пример использования: python -m src sources

3) read - Читает и показывает задачи из указанных источников с возможностью фильтрации, имеет несколько опций и аргументов:
* В качестве аргументов можно указать файлы формата jsonl, тогда задачи будут читаться из них

  В самом репозитории уже есть 2 jsonl файла (example_1.jsonl и example_2.jsonl), с котрыми вы можете протестировать работу программы самостоятельно.

  Примеры:
  ```
  python -m src read example_1.jsonl
  ```
  ```
  python -m src read example_1.jsonl example_2.jsonl
  ```

* --stdin - Считывать задачи из сдандартного ввода

  Задачи нужно писать в формате ```id:title:author:content```. Если вы захотите перестать вводить задачи, напишите stop.

  Пример использования:
  ```
  python -m src read --stdin
  ```
  Дальше вводим задачи, например: ```my_id:my_title:my_author:my_content```
  
* --max_tasks - Установить максимум, сколько задач может быть прочитано

  По умолчанию 10, если поставить 0, то можно прочитать неограниченное число задач. При одновременном вводе --max_tasks 0 и --stdin не забывайте про возможность завершить ввод задач командой stop.

  Примеры:
  ```
  python -m src read --stdin --max-tasks 5
  ```
  ```
  python -m src read example_1.jsonl --max-tasks 0
  ```

* --author - Считывать задачи только указанного автора

  Все встреченные задачи с другим автором будут проигнорированы и не будут считываться программой.
  
  Пример:
  ```
  python -m src read --stdin --author milena
  ```
  
* --contains - Считывать только задачи с указанной подстрокой в содержании

  Пример:
  ```
  python -m src read example_1.jsonl --contains проект
  ```
  Задача '8:Сделать бэкап:milena:Сделать бэкап базы данных проекта' будет считана.
  Задача '9:Написать тесты:cool_user:Написать тесты к лабе по python' не считается.
  
* Все аргументы и опции можно комбинировать, примеры:
  ```
  python -m src read example_1.jsonl example_2.jsonl --max-tasks 20 --author kate
  ```
  ```
  python -m src read example_1.jsonl example_2.jsonl --stdin --max-tasks 20
  ```
  ```
  python -m src read example_1.jsonl --stdin --author alina --contains python
  ```
